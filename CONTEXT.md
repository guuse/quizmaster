## What it is

A live, host-paced multiplayer quiz game (Kahoot-style) where the quiz is **generated by
Claude** and **nobody â€” not even the creator â€” sees the answers**. A creator generates a
quiz from a topic, shares a link, and everyone (creator included) plays together in real
time: synchronized questions, a countdown, an answer reveal, and a live leaderboard.

## Core model decisions

| # | Decision | Choice |
|---|----------|--------|
| 1 | Game type | **Live host-paced** (synchronized, server-driven), not async self-paced. |
| 2 | Host model | **Server hosts.** No privileged human. The creator plays like everyone else; the server is the single source of truth and paces the game on a timer. |
| 3 | Stack | **TypeScript everywhere.** Socket.IO over WebSockets. Single stateful Node server. React + Vite SPA. Claude called server-side only. |
| 4 | Identity | **Accounts for creators (Google OAuth), anonymous nickname-only for players.** |
| 5 | Persistence | **SQLite + Prisma.** Durable: `users`, `quizzes` (questions+answers), `game_results`. In-memory only: live room state. |
| 6 | Auth | **Google OAuth** (single provider). Session cookie + `sessions` table. No passwords, no reset email. |

## Quiz generation

- **Creator inputs:** topic (free text), question count (5â€“20), difficulty (easy/medium/hard),
  timer per question (10/20/30s).
- **Question shape:** multiple choice with **4 options**, or **true/false** (2 options).
  Unified as `{ text, options: string[2|4], correctIndex }`. Same UI grid, same scoring.
- **No editing.** Generate â†’ quiz is sealed â†’ creator joins as a player. Preserves
  "nobody has seen the answers."
- **Reliability:** force structured output via a `submit_quiz` tool (`tool_choice`), then
  **validate server-side** (exactly one in-range `correctIndex`, right option count, no empty
  or duplicate options). Retry once on failure, then surface an error. Never persist unvalidated.
- **Model:** **Claude Sonnet 5** (`claude-sonnet-5`) â€” factual reliability matters because a
  wrong answer key is undetectable (nobody sees the answers).
- **Timing:** synchronous generation with a loading state. No job queue.
- **Auth-gated:** only a logged-in creator can trigger a Claude call (cost/abuse guard).

## Game loop (server-authoritative)

1. **Lobby** â€” players join via link, pick nicknames, appear in a live list. Creator sees **Start**.
2. **Question** â€” server pushes text + options + a server-authoritative countdown (creator-chosen 10/20/30s).
3. **Close** â€” on timer hitting 0 or when everyone has answered, whichever first.
4. **Reveal** (~5s) â€” correct answer + **answer distribution** (bar per option) + per-player "+points".
5. **Leaderboard** (~5s) â€” cumulative standings.
6. Loop to next question.
7. **Final results** â€” podium/winner. Write one `game_results` row.

- **Start** is the creator's only privileged action (not a pacing advantage). After Start, the
  server paces everything.
- **Scoring:** speed-scaled (Kahoot formula, `round(1000 * (1 - (timeUsed/timeLimit)/2))`),
  computed server-side from the received-answer timestamp. Wrong = 0. No streak bonuses (MVP).

## Rooms, invites, reconnection

- **Room code** (~6 chars) + shareable link `.../join/<CODE>`. Both resolve to one join.
- **Join window:** lobby-only. Locked at Start. No mid-game late-join (MVP).
- **Replayable:** a saved quiz spins up a fresh room (new code) each play; each writes its own
  `game_results` row.
- **Reconnection:** on join the server issues a `playerToken` stored in `localStorage`; on
  reconnect the client sends it back and the server re-syncs current state + score. Socket.IO
  handles transport reconnect; the token handles identity.
- **Game never waits** on a disconnected player â€” they get 0 for questions that close while away.
- **Creator drop:** harmless mid-game (server paces). Pre-Start it just stalls the lobby; **no
  host-migration** (MVP).
- **Reaper:** a room with no connected players for ~2 min is deleted from memory.
- **Concurrent rooms:** many independent rooms in one process (`Map<roomCode, GameState>`).

## Deployment (home k3s + Argo CD)

Target is the `guuse/homeserver` single-node k3s cluster, deployed via Argo CD app-of-apps.

- **Onboard:** add `homeserver/argocd/applications/quizmaster.yaml` â€” an `Application` â†’
  `guuse/quizmaster` repo `deploy/` path (Kustomize), `project: homeserver`, auto-sync
  prune+selfHeal, `CreateNamespace=true`.
- **Build:** GitHub Actions on push to `main` builds a multi-stage Dockerfile and pushes
  `ghcr.io/guuse/quizmaster:<sha>`. **Argo CD Image Updater** (CR, newest-build, SHA-tag regex)
  writes the tag into `deploy/kustomization.yaml` â†’ auto-rollout. Self-contained workflow
  (not the Harborn org action).
- **Workload shape:** `Deployment` **`replicas: 1` + `strategy: Recreate`** (SQLite file + in-
  memory game state cannot be shared). `local-path` **PVC** mounts the SQLite file. `Service`
  + `Ingress` host `quizmaster.guuse.online`, TLS via the wildcard TLSStore (no per-app cert).
- **Secret:** the Claude credential as a Kubernetes `Secret` in the `quizmaster` namespace,
  applied out-of-band (never committed), same pattern as `cloudflare-token`. The server accepts
  either `ANTHROPIC_API_KEY` (SDK `apiKey`) or `CLAUDE_SDK_TOKEN`/`ANTHROPIC_AUTH_TOKEN` (an
  `sk-ant-oat01-â€¦` OAuth token â†’ SDK `authToken` + oauth beta header). The repo `.env` currently
  holds `CLAUDE_SDK_TOKEN` (OAuth token), so that path is the live one.

### Reachability â€” Tailscale Funnel via the operator

The cluster is tailnet-only, but the app must be reachable by non-tailnet friends. Decision:
**Tailscale Funnel**, attached via the **Tailscale Kubernetes operator**.

> This reverses the documented "operator intentionally NOT used" decision in
> `homeserver/AI_CONTEXT.md`. **That file must be updated** to record the operator is now in
> use and why. The operator becomes an Argo CD-managed component
> (`argocd/applications/tailscale-operator.yaml`).

- Friend-facing URL is a `*.ts.net` Funnel hostname (e.g. `quizmaster.tail06b729.ts.net`) â€”
  Funnel cannot serve the custom `guuse.online` domain. `quizmaster.guuse.online` stays
  tailnet-internal (for the creator on the tailnet).
- **Google OAuth callback** must register the `*.ts.net` Funnel origin.

## UX defaults

- Mobile-first (players are on phones); desktop scales up.
- Reveal shows answer distribution + correct answer + per-player points.
- Per-quiz config kept minimal: question count + timer only.
